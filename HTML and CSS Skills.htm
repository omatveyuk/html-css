HTML and CSS Skills

HTML and CSS Skills is meant to help you practice and assess your understanding of HTML and CSS.

Follow the directions below to review what you should have learned, practice implementing HTML and CSS, and submit an assessment of your understanding.

Take Aways

  • know how to build basic HTML pages and forms
  • know what HTML requirements you need for a GET vs. POST request
  • know how to create and integrate CSS stylesheets to stylize your HTML
  • know how to make your HTML accessible

Assessment

Complete your HTML and CSS code in the application-form.html and styles.css files.

You must turn in all of your work through FRODO by 9PM on Sunday.

Directions

Part 1: HTML

Add to the HTML document application-form.html so that it has the following items.

  1. A form that asks the user:
    • their first and last name (as separate fields),
    • their salary requirement as a number, and
    • which job they are applying for, from a list of:
      • Software Engineer
      • QA Engineer
      • Product Manager
  2. The form should also have a submit button that submits the form to the URL path /application-success. This should be a POST request. (For now, just focus on the HTML requirements for this request. Don’t build a Flask route, yet.)
  3. The page should have a large heading and some explanatory text, as well.
  4. Add the elements to make your HTML form accessible to screen readers. For more information, see http://webaim.org/techniques/forms/controls.

Part 2: CSS

Open the CSS file, styles.css, and use this to stylize the HTML file, application-form.html.

First, edit the HTML file to reference the CSS file. Then, complete each of the following styles using CSS:

  • Change the background color of all of the form input boxes and submit button to be yellow. (Can you do this for all input boxes without having to stylize each one independently?)
  • Change the text in the last name field to be bold.
  • Draw a border around the instruction box.
